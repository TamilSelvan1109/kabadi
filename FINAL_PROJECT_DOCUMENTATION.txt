INSTALLATION STEPS:

USAGE WORKFLOW:
==============
Step 1: Boundary Setup
- Run: python main.py â†’ Option 1
- Choose detection method (Two Points recommended for beginners)
- Draw boundary line on video frame
- Click SAVE to store configuration

Step 2: Start Tracking
- Run: python main.py â†’ Option 2
- System loads boundary configuration automatically
- Real-time tracking begins immediately
- Press 'q' to quit

Step 3: Review Evidence
- Check violations/screenshots/ for violation images
- Check violations/videos/ for violation recordings
- Files are automatically timestamped and organized

CONFIGURATION OPTIONS:
=====================
Video Sources (video_config.py):
- Change VIDEO_PATHS dictionary for different input sources
- Set to 0 for webcam, or path for video files
- Configure RTSP URLs for IP cameras

Detection Parameters:
- YOLO confidence: Adjust in yolo_detector.py (default: 0.5)
- MediaPipe confidence: Modify in skeleton_tracker.py (default: 0.3)
- Player matching distance: Change in player_id_manager.py (default: 150px)

Performance Tuning:
- Frame resolution: Modify target_width in player_tracker.py
- Buffer size: Adjust in violation_recorder.py (default: 150 frames)
- Cleanup threshold: Change max_frames_missing (default: 60 frames)

TROUBLESHOOTING GUIDE:
=====================
Common Issues:

1. "No module named 'cv2'":
   Solution: pip install opencv-python==4.8.1.78

2. "MediaPipe model not found":
   Solution: Download pose_landmarker_lite.task to models/ directory

3. "YOLO model download failed":
   Solution: Check internet connection, manually download yolov8n.pt

4. "Video file not found":
   Solution: Verify video path in video_config.py, check file exists

5. "Low FPS performance":
   Solution: Reduce video resolution, close other applications, consider GPU

Performance Optimization:
- Use SSD storage for faster video I/O
- Close unnecessary applications during processing
- Use GPU acceleration if available
- Reduce video resolution for better performance
