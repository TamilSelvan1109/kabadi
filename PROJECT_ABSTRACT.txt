===============================================================================
                              PROJECT ABSTRACT
===============================================================================

OVERVIEW:
This project presents an advanced computer vision-based system for real-time 
Kabaddi player tracking and boundary violation detection, designed to modernize 
sports officiating through AI technology. The system integrates cutting-edge 
pose estimation, multi-player tracking, automated violation detection, and 
cloud-based logging to provide accurate, real-time analysis of Kabaddi matches.

===============================================================================
                           TECHNICAL ARCHITECTURE
===============================================================================

CORE TECHNOLOGIES:
• Computer Vision Framework: OpenCV 4.8+ for image processing and video analysis
• AI/ML Engine: MediaPipe 0.10+ with 33-landmark pose estimation model
• Web Interface: Flask-based real-time dashboard with HTML5 Canvas
• Backend Processing: Python with NumPy for mathematical computations
• Real-time Streaming: HTTP video streaming with frame-by-frame analysis
• Cloud Database: MongoDB Atlas / Firebase for violation log storage
• Live Streaming: GoPro camera integration via RTMP/HTTP protocols

SYSTEM COMPONENTS:

1. ADVANCED POSE DETECTION (foot_detector.py)
   • 33-Point Skeleton Tracking: Full-body pose estimation using MediaPipe
   • Multi-Landmark Foot Analysis: Heel (29,30), toe (31,32), ankle (27,28)
   • Confidence-Based Filtering: Only processes landmarks with >30% confidence
   • Ground Contact Point Detection: Precise foot-ground interaction points

2. INTELLIGENT PLAYER MANAGEMENT (player_tracker.py)
   • Dynamic Player Assignment: Manual click-to-assign player identification
   • Multi-Player Support: Tracks up to 8 players simultaneously
   • Unique ID Management: Persistent P1, P2, P3... identifiers
   • Proximity-Based Selection: 50-pixel radius click detection

3. SMART BOUNDARY DETECTION (line_detection.py)
   • Three Detection Methods:
     - Two-Point Manual: Simple start-end point definition
     - Multi-Point Manual: Complex boundary shapes with multiple vertices
     - Hough Transform Auto-Detection: AI-powered line detection
   • Interactive Web Interface: Real-time boundary drawing
   • Scalable Coordinates: Automatic scaling between resolutions

4. VIOLATION DETECTION ENGINE (violation_detector.py)
   • Real-time Boundary Crossing: Pixel-perfect foot position analysis
   • Multi-Violation Tracking: Counts violations per player (3-strike elimination)
   • Evidence Collection: Automatic screenshot capture with timestamps
   • Temporal Filtering: 2-second cooldown to prevent duplicates

5. CLOUD DATABASE INTEGRATION (cloud_logger.py)
   • MongoDB Atlas: Scalable cloud database for violation logs
   • Firebase Integration: Real-time data synchronization
   • Automatic Backup: Redundant storage across multiple cloud providers
   • Analytics Dashboard: Historical data analysis and reporting

6. LIVE WEB DASHBOARD (app.py + live_dashboard.html)
   • Real-time Video Streaming: 30 FPS live processing with overlays
   • Interactive Setup: Drag-and-drop video upload with boundary config
   • Live Statistics: Active players, violations, elimination tracking
   • Professional UI: Gradient design with responsive layout

===============================================================================
                        SKELETON COLOR CODING SYSTEM
===============================================================================

PLAYER STATUS VISUALIZATION:
• GREEN SKELETON (RGB: 0,255,0): Active player, no violations
• YELLOW SKELETON (RGB: 255,255,0): Warning state, approaching boundary
• ORANGE SKELETON (RGB: 255,165,0): First violation detected
• RED SKELETON (RGB: 255,0,0): Multiple violations, critical state
• DARK RED SKELETON (RGB: 139,0,0): Player eliminated (3+ violations)
• WHITE SKELETON (RGB: 255,255,255): Unassigned/detected player
• BLUE SKELETON (RGB: 0,0,255): Player under manual review

SKELETON COMPONENTS:
• Thick Lines (3px): Violation state indicators
• Medium Lines (2px): Active player tracking
• Thin Lines (1px): Background/unassigned players
• Highlighted Joints: Key landmarks (shoulders, hips, feet)
• Foot Markers: Yellow circles around foot landmarks for precision
